Design a mobile react native app UI called “MatchA”.

PRODUCT CONCEPT
MatchA is a self-improvement mobile app for single men aged 27–38 who want to understand which personal, physical, style, habit, and social factors they could improve to increase their chances of getting a date.

The app should combine:
- profile discovery inspired by Tinder
- personal development / self-improvement
- progress tracking for attractiveness, presence, habits, and social skills

This is NOT a traditional dating app focused on messaging.
There must be NO chat, inbox, or conversation UI.
The experience should focus on:
1. exploring curated profiles,
2. learning from patterns across attractive profiles,
3. translating those insights into concrete personal goals.

LANGUAGE
The app must support:
- Spanish as the primary UI language
- English as a secondary language option
- automatic localization based on device language

Please provide all key interface text in both Spanish and English.

TARGET AUDIENCE
- Single men aged 27–38
- Interested in self-improvement to improve their dating prospects
- Expect a premium, modern, mobile-native experience

PLATFORM
- Mobile only
- Designed primarily for smartphone screens
- Responsive and touch-friendly
- Clean, scrollable layouts optimized for mobile use

VISUAL STYLE
The design should feel:
- modern
- minimalist
- clean
- premium
- masculine, but elegant and soft
- aspirational
- visually pleasing and emotionally engaging
- to have a green like theme nature oriented

Avoid making it feel:
- like a generic dating app
- overly flashy
- childish or gimmicky
- too cold or corporate
- cluttered or heavy

APP STRUCTURE
The app must have ONLY 4 functional views:
1. Login
2. Profile
3. Discover
4. My Goals

Important:
Even if multiple static screens are shown, they must still map to only these 4 functional views.

In particular:
- Discover is only ONE functional view
- It must be presented in 3 static visual states:
  - card stack
  - like
  - dislike

These are not separate views, only different visual states of the same Discover screen.

NAVIGATION
Use a bottom tab bar with ONLY 3 tabs:
- Discover
- Profile
- My Goals

Settings must NOT be a separate tab.
Settings should instead appear as a small action, icon, or entry point inside the Profile screen.

REQUIRED DELIVERABLE
Produce exactly these screens:
- 1 Login screen
- 1 Profile screen
- 3 static representations of the Discover view:
  - stack state
  - like state
  - dislike state
- 1 My Goals screen

Even though 6 static screens are shown, the product still has only 4 functional views:
- Login
- Profile
- Discover
- My Goals

SCREEN REQUIREMENTS

1) LOGIN SCREEN
Create 1 Login screen.

It should look like a premium onboarding/login experience and include:
- MatchA logo or wordmark
- aspirational or lifestyle background image
- strong welcome headline
- primary CTA to create account
- secondary CTA to sign in
- small legal/disclaimer text
- emotional but clean visual composition

The feeling should be:
- premium
- aspirational
- intimate but not sentimental
- modern
- elegant

This screen should visually communicate:
- confidence
- self-improvement
- modern masculinity
- premium mobile product

2) PROFILE SCREEN
Create 1 editable Profile screen.

It should feel like a polished, detailed profile/settings editor and include:
- header: “Mi Perfil / My Profile”
- photo section with up to 10 images
- horizontal image thumbnails
- add photo button/action
- full name field
- age field
- editable physical attributes with dropdown/select inputs:
  - body type
  - hair color
  - height
  - weight
  - ethnic background
- multi-select interest chips/tags
- short bio/about me field
- primary CTA: “Guardar cambios / Save changes”
- small Settings entry or icon inside this screen

Please propose realistic and modern default option lists for:
- physical attributes
- interests

The screen should be long, editable, well-structured, and easy to scan on mobile.

3) DISCOVER VIEW
Create 3 static mockups of the SAME Discover view. This should have a few dummy data females options that have metadata to train yourself and give feedback on the goals after liking an specific profile.

GENERAL REQUIREMENTS FOR DISCOVER
The Discover screen must include:
- top bar or header
- main profile card with large image
- visible name, age, location, and selected attributes
- Tinder-inspired card design
- lower interaction buttons
- bottom navigation visible

It must NOT include:
- chat
- messaging
- conversation previews
- inbox patterns

The Discover experience must feel like profile exploration with a self-improvement lens, not a classic dating app.

3A) DISCOVER — STACK STATE
Show:
- the main front card
- hints of the next cards behind it
- dislike and like buttons below
- premium, clean card-stack interaction

3B) DISCOVER — LIKE STATE
Show the same screen in a positive interaction state:
- same card or equivalent transition
- visible positive overlay, badge, or stamp
- clear approval feedback
- keep the same navigation and layout structure

3C) DISCOVER — DISLIKE STATE
Show the same screen in a negative interaction state:
- same card or equivalent transition
- visible rejection/discard overlay, ideally using red or a similar visual cue
- clear rejection feedback
- keep the same navigation and layout structure

4) MY GOALS SCREEN
Create 1 My Goals screen.

It should feel like a premium personal progress dashboard, not a basic list.

It must include:
- header: “Mis Metas / My Goals”
- top summary area with:
  - overall progress percentage
  - number of active goals
- category filters such as:
  - Todas / All
  - Físicas / Physical
  - Personalidad / Personality
- list of active goal cards

Each goal card must include:
- goal title
- category
- current completion percentage
- progress bar
- suggested mini-task or next action
- expected impact if completed
- visual treatment inspired by a checklist or personal improvement plan

Use example goals such as:
- Resistencia cardiovascular / Cardiovascular endurance
- Confianza social / Social confidence
- Higiene y cuidado / Hygiene and grooming
- Inteligencia emocional / Emotional intelligence
- Postura corporal / Body posture
- Habilidades de charla / Conversation skills

CONTENT TO PROPOSE
Please create and define:
- realistic default dropdown values for physical attributes
- modern, believable interest tags
- sample profile content for Discover cards
- sample goals with progress percentages and suggested actions
- premium microcopy for buttons, headings, empty states, helper text, and encouragement messages

DESIGN SYSTEM GUIDANCE
Also define:
- visual hierarchy
- UI components
- button styles
- input styles
- card styles
- spacing logic
- bottom navigation behavior
- icon style
- typography direction
- color direction
- recommendations to make the product feel premium, coherent, and mobile-native

TEXT REQUIREMENT
For all important interface elements, provide text in both:
- Spanish
- English

FINAL DESIGN INTENT
The final result should stay as close as possible to this structure:
- visual, aspirational login
- long editable profile with attributes and photos
- Discover shown in three states of the same functional view
- My Goals as a detailed progress dashboard with improvement cards

The final design must feel premium, modern, emotionally engaging, and clearly differentiated from a standard chat-based dating app.
